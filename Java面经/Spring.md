### IOC&AOP

#### IOC

思想：

IoC 的思想就是将原本在程序中手动创建对象的控制权，**交由 Spring 框架来管理**。不过， IoC 并非 Spring 特有，在其他语言中也有应用。

**为什么叫控制反转？**

- **控制** ：指的是对象创建（实例化、管理）的权力
- **反转** ：控制权交给外部环境（Spring 框架、IoC 容器）

#### AOP

AOP可以做那些与项目无关，却是业务逻辑共同调用的责任，**减少重复的代码，降低耦合度**
